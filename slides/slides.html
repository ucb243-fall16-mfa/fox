Multiple Factor Analysis in R Using "MFA" Package

  • This presentation will give a brief overview of how and when to use Multiple Factor Analysis.

  • We will also demonstrate how to do Multiple Factor Analysis using the "MFA" package in R, using a fictitious example.

  • The source for this paper is "Multiple factor analysis: principal component analysis for multitable and multiblock data sets" by Herve Abdi, Lynne J. Williams, and Domininique Valentin.

What is Multiple Factor Analysis, and when should you use it?

  • Multiple Factor Analysis (MFA) is used to analyze different sets of data taken on the same observations. These data sets can include different types and numbers of variables.

  • The goal is to determine any agreement or disagreement between the data sets. For instance, in the example we walk through in later slides, we see different wine experts rate an assortment of wines on 4 different criteria. Through MFA, we can see how much the experts agree or disagree on each of those criteria.

  • One output of MFA is a set of factor scores, which are coordinates that can be used to plot each of the observations. This plot will show how similar the observations are to each other based on their distance from each other on the plot.

  • The analysis also produces partial factor scores for each factor, which contain the relative position of the other factors.

What is MFA? (cont.)

  • The main piece of MFA is to conduct a principle component analysis (PCA) on the combined (normalized) data sets. This produces a new set of variables, called components, with each component explaining a portion of the total variance.

  • Each of the original variables contributes to each component. The amount a variable contributes to a certain component is called the loading of the variable on that component and is determined through MFA.

  • Using the factor scores and loadings, we can determine the contribution of each observation, variable and data table to each component. The observation/variable/table with the highest contribution becomes most important in understanding that component.

  • Finally, to compare two tables to one another we can compute the \(R_V\) coefficient and the \(L_g\) coefficient, which reflect the similarity between the two tables.

Overview of the Steps of MFA

  • There are multiple ways to perform MFA on a given data set (i.e. using generalized singular value decomposition, simple singular value decomposition on a transformed data set, cross-product matrices, or using the factor scores from a single data table).

  • The "MFA" package in R uses a simple singular value decomposition on a transformed data set, so that is the method we will walk through in this presentation. For others, see the cited paper.

Overview of the Steps of MFA

  • Start with a data set \(\mathbf{X}\) containing \(J\) variables measured on a common \(I\) observations, separated into \(K\) data tables \[ \mathbf{X} = [\mathbf{X_{[1]}} | ... | \mathbf{X_{[K]}}] \]
    where each matrix \(\mathbf{X_{[i]}}\) has \(I\) rows.

Normalize the Data

  • Center each column by its mean
  • Scale each column such that the sum of the squares of its entries is equal to 1

Calculate the weight of each data table

  • Run a simple PCA on each data table
  • \(\gamma_i\) is the first singular value for table \(i\)
  • The weight for table \(i\) is \[ \alpha_i = \frac{1}{{\gamma_i}^2} \]

Create the weightings matrix A

  • Calculate the weight for each table and store in a matrix, \(\mathbf{A}\) \[ \mathbf{A} = \mbox{diag}\{ \alpha_1\mathbf{1_{[1]}}, \alpha_2\mathbf{1_{[2]}}, ..., \alpha_K\mathbf{1_{[K]}}\} \]
    where \(\mathbf{1_{[k]}}\) is a vector of ones the length of the number of variables in table \(\mathbf{X_{[k]}}\)

Transform the original data set using the weightings matrix A

  • Scale each data table by the square root of its weight and concatenate into a new, transformed data set \(\mathbf{\tilde{X}}\) \[ \mathbf{\tilde{X}} = [\sqrt{\alpha_1}\mathbf{X_{[1]}} | \sqrt{\alpha_2}\mathbf{X_{[2]}} | ... | \sqrt{\alpha_K}\mathbf{X_{[K]}}] \]

Simple PCA on Transformed Data

  • Singular value decomposition of transformed data gives \[ \mathbf{\tilde{X}} = \mathbf{\tilde{U}}\mathbf{\tilde{\Delta}}\mathbf{\tilde{V}}^T \]
  • From this, we can calculate the factor scores as well as the loadings.

Calculating the Factor Scores and Loadings

  • Factor Scores \[ \mathbf{F} = \mathbf{\tilde{U}}\mathbf{\tilde{\Delta}} \]
  • Loadings \[ \mathbf{Q} = \mathbf{\tilde{V}}\mathbf{A}^{-1/2} \] where \(\mathbf{A}\) is the matrix of the weights.

Calculating the Partial Factor Scores

  • First obtain partial loadings for factor \(i\), \(\mathbf{Q_{[i]}}\), using the loadings table \(\mathbf{Q}\) \[ \mathbf{Q} = [\mathbf{Q_{[1]}}^T | ... | \mathbf{Q_{[K]}}^T]^T \]
  • Partial factor scores for factor \(i\) \[ \mathbf{F_{[i]}} = K * \alpha_i * \mathbf{X_{[i]}}\mathbf{Q_{[i]}} \] where \(K\) is the total number of factors in the analysis, \(\mathbf{X_{[i]}}\) is the original data table, and \(\mathbf{Q_{[i]}}\) is the matrix of partial loadings for that factor

The mfa function

mymfa <- mfa(data, sets, ncomps = NULL, center = TRUE, scale = TRUE)

Creates object of class "mfa" containing

  1. eigenvalues for the original data set

  2. common factor scores

  3. partial factor scores

  4. loadings matrix

Inputs to the mfa function

  • "data": data frame or a matrix
  • "sets": list of vectors of length equal to the number of data tables. Each vector contains variables from the original data in that table. The vectors can either be numeric (column indices) or characters (column names)
  • "ncomps": an integer or NULL value, specifying the number of components you want in the output of the MFA
  • "center": either a logical value or a vector of length equal to the total number of active variables, specifying the value around which each column will be centered. If "center" is TRUE, each column will be centered around its mean. If "center" is FALSE, no centering will be done.
  • "scale": either a logical value or a vector of length equal to the total number of active variables, specifying the value by which each column will be scaled. If "scale" is TRUE, each column is scaled by its standard deviation. If "scale" is FALSE, no scaling will be done.

Contributions of an Observation to a Dimension

contrib_obs(mymfa)
# mymfa is an object of class 'MFA'

\[ ctr_{i,l} = \frac{(m_{i}\times f_{i,l}^2)}{\lambda_{l}}\]

  • i, observation index
  • l, dimension index
  • \(m_{i}\), mass of the i-th observation
  • \(f_{i,l}\), factor score of the i-th observation for the l-th dimension
  • \(\lambda_{l}\), eigenvalue of dimension l

Contributions of a Variable to a Dimension

contrib_var(mymfa, sets, tables)
# sets is the list of the variables we will study
# tables are the original tables

\[ ctr_{j,l} = a_{j}\times q_{j,l}^2\]

  • j, variable index
  • l, dimension index
  • \(a_{j}\), weight for the j-th variable
  • \(q_{j,l}\), loading of the j-th variable for the l-th dimension

Contributions of a Table to a Dimension

contrib_tables(mymfa, sets, tables)

\[ ctr_{k,l} = \sum_{j=1}^{J_{k}} ctr_{j,l}\]

  • k, table index
  • l, dimension index
  • Sum is over the variables of table k
  • \(ctr_{j,l}\), the contribution of variable j to dimension l

Between-Table Structure

RV(table1, table2)
RV_table(dataset, setsarg = list(1:2, 3:4))
# dataset is a dataframe with numeric variables

\[\scriptsize{RV_{k,k'} = \frac{ Tr((X_{[k]}.X_{[k]}^{T})\times (X_{[k']}.X_{[k']}^{T}))} {\sqrt{(Tr((X_{[k]}.X_{[k]}^{T})\times (X_{[k]}.X_{[k]}^{T})) \times Tr((X_{[k]}.X_{[k]}^{T})\times (X_{[k]}.X_{[k]}^{T})))}}}\]

  • \(X_{[k]}\), first table of data
  • \(X_{[k']}\), second table of data
  • Tr, trace operation

Lg Coefficient

The LG coefficient between two tables is defined as:

\[ L_{g(k,k')} = Tr[(X_kX_k^T) \times (X_{k'} X_{k'}^T)] \times (\alpha_k \times \alpha_{k'}) \]

where \(\alpha_k\) is the inverse of the largest singular value squared of \(X_k\).

The function Lg computes the Lg coefficient between two tables. It takes as inputs two tables or matrices, and outputs a scalar.

library(MFA)
Lgtable1_table2 <- Lg(tables[[1]], tables[[2]])
Lgtable1_table2
## [1] 0.9180402

Bootstrap Procedure

Bootstrap for the factor scores is for estimating the stability of the compromise factor scores.

First, we sample from the table index. If the dataset is combined with k tables, we sample from 1 to k for k times, which becomes our first sample set. Second, we compute the partial factor scores for our sample sets, and we denote it as \[F_1,\dots,F_k\] and denote \[F_1^*=\frac{1}{k}\sum_{i=1}^k F_i\] as the bootstrapped estimate of the factor scores from the first sample.

Bootstrap Procedure Continued

Repeat step 1 and 2 for large number of times (e.g., L=1000), so we can get \[F_1^*,F_2^*,\dots,F_L^*\]

Then we can calculate the mean and variance of bootstrap samples: \[\bar{F}^*= \frac{1}{L}\sum_{l=1}^L F_l^*\] \[\hat{\sigma}_{F^*}^2=\frac{1}{L}(\sum_{l=1}^{L}( F_l^*-\bar{F}^*)\cdot(F_l^*-\bar{F}^*))\]

Finally, we can get the bootstrap ratio by \(\frac{\bar{F}^*}{\hat{\sigma}_{F^*}}\). The bootstrap ratio gives information of which observations most reliably contribute to a component.
